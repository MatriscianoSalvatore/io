Mobile Developer at Deloitte, Milano

I've done a 6 months internship working as a Mobile Developer for Deloitte in Milano from 17/02/2020 to 17/08/2020.
The firts project I worked on was the Android native app of a prestigious italian automotive manufacturer.
I really liked it because it was a very cool and famous car brand and a very nice app wit cool features. The main purpose of the app was to control the connected car, so open and close it, switch the lights and the air conditioning on and off etc, everything via app. I worked at fixing some bugs, as fixing the Facebook login that was not working anymore, fixing the map view for the geofencing that was not displaying correctly etc.
For this project I used Java, Android Studio, Git, Bitbucket, Fork and Microsoft Planner and I focused my attention and studied how Design Patterns can be applied to Android apps and the differences between MVC, MVP and MVVW, how Clean Architecture can be applied to Android and a little bit about Kotlin.
Unfortunately, after the first week at the office, I had to work remotely from home because of the restrictions for the Covid-19 emergency, so I had a lot of meetings and videocalls using Skype, Zoom and Microsoft Teams.

The second project I worked on was the iOS and Android cross-platomfm app in Ionic (Angular) for an energy provider.
I liked it because it was the first time I worked with Ionic and the first time I made something for iOS (excepet for a very little experiment with React Native a few years ago) so it was quite challenging and stimulating.
One of the features I worked on is the realization of the whole notifications section, so the layout of the page, the integration of the notifications api with the implementation of the get function to get the whole list of the user's notifications from backend and the put to mark a notification as read, unread or deleted, I ordered the notifications in the list for prioririty and date and I made the tooltip component with the action buttons.
The second feature I worked on is the news section, in particular I made the Article Detail Page, with the sharing component and the progress bar informing the user about the percentage of the article's reading, auto-filling itself with the scrolling of the page.
An other feature I worked on is the app login via fingerprint and face recognition.
Then I worked at the architecture review of the whole app and re-made a lot of pages of the app completely modifying and refactoring both the logics and the graphical layout of the pages I worked on.
For this project I used Visual Studio Code, Git, GitLab, Fork, Jira, Zeplin and Confluence.
The work was organized in Agile Scrum methodology with 2 weeks sprints and I partecipied at daily standup meetings and weekly product backlog refiniments, sprint revies, sprint plannings, sptint retrospectives, user stories effort estimations etc.
